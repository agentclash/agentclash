A **eval pack** is the versioned YAML bundle that defines a benchmark task: its prompts, tools, sandbox, evaluation spec, and input cases. These pages complement the short concept guide [Eval packs and inputs](../concepts/eval-packs-and-inputs). They spell out everything a benchmark author needs to publish a pack that survives server-side parsing, validation, and execution.

Everything here is keyed to shipped code paths—not roadmap language. When behavior changes upstream, validate again with:

```bash
agentclash eval-pack validate your-pack.yaml
```
